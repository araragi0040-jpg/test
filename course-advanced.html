この教材 01 / 03　解説（WIKI）
所要時間 約15分
前提 研修用Microsoft 365アカウントと研修サイトURLを受け取っていること
次 02 練習問題（35分）→ 03 確認問題（10分）

この章のゴール

  • 正しい研修用アカウントでログインできる
  • 指定されたSharePointサイトを直接URLから開ける
  • アカウント、サイト名、URL、作業権限を確認できる
  • 閲覧画面と編集画面の違いを説明できる
  • サイトコンテンツ、サイトページ、リスト、ライブラリを開ける
  • ページを編集し、下書き保存して再度開ける
  • 下書き、公開、再発行、変更破棄の違いを説明できる
  • リスト項目とファイルの登録・編集・削除を安全に行える
  • 操作できない場合に、アカウント・URL・権限を順番に確認できる

操作を始める前に、アカウント・サイト・権限・公開状態を確認する。

パスワードや認証コードは提出物へ記載しません。

2．サイトへの入り方

基本ルート

研修では講師指定の直接URLを使います。

ブラウザを開く
→ 研修サイトURLを入力
→ 研修用アカウントでログイン
→ 必要に応じて多要素認証
→ 研修サイトを開く

開始画面からサイトを探すより、作業対象を間違えにくい方法です。

補助ルート

Microsoft 365ホーム
→ アプリ起動ツール
→ SharePoint、Sites、サイトのいずれか
→ 研修サイトを選択

表示名や位置は組織・更新状況で異なる場合があります。

複数アカウントを使っている場合

  • 右上のプロフィールで現在のアカウントを確認する
  • ブラウザプロファイルを研修用に分ける
  • InPrivateまたはシークレットウィンドウを使う
  • 別組織の画面が開いた場合はサインアウトして入り直す

3．サイトを開いた直後の4確認

1. 右上のログインアカウント
2. サイト名
3. ブラウザのURL
4. 自分の作業権限

更新者として演習する場合、ページの「編集」、リストの「新規」、ライブラリの「アップロード」が表示されるか確認します。

表示されない場合、自分で権限を変更せず講師へ連絡します。

Microsoft Lists 案件や申請の一覧管理
設定 サイト設定・権限などの確認

サイトの種類と権限によって表示は異なります。

5．閲覧画面と編集画面

閲覧画面

  • ページを読む
  • リンクを開く
  • ニュース、案件、資料を見る

編集画面

  • セクションを追加する
  • Webパーツを追加・移動・削除する
  • テキスト・画像・リンクを設定する
  • 下書き保存、公開、再発行、変更破棄を行う

6．下書き・公開・再発行・変更破棄

操作 結果
下書きとして保存 変更を保存する。通常は一般閲覧者へまだ反映しない
公開 新しいページをサイト利用者へ表示する
再発行 公開済みページの変更をサイト利用者へ反映する
変更を破棄 現在の編集内容を保存せず戻す

講師の指示がない限り、第0章では公開・再発行を行いません。

3．FormsとListsの違い

Microsoft Forms

Formsは、回答者から同じ形式で情報を集めることに向いています。

例：

  • アンケート
  • 改善アイデア
  • 問い合わせ
  • 研修申込

回答者には、必要な質問だけを表示できます。

Microsoft Lists

Listsは、登録後も内容を更新しながら一覧管理することに向いています。

例：

  • 案件進捗
  • 問い合わせ対応
  • 備品管理
  • 課題管理

担当、期限、ステータス、優先度などを一覧で確認できます。

判断の目安

回答を集めることが中心
→ Forms

登録後も状態や数値を更新する
→ Lists

問題5
